# Projekt Planlægning

Eventuelt inden projekt: teori og øvelser om [polymorfi](polymorfi.md)

---

**Mål:** Udvikle et program der kan anvendes til planlægning.

- Der **skal** gøres brug af polymorfi.
- Grafisk brugerflade er ikke et krav.

Projekteksempler:

- Analogt/digitalt ur og tidsfunktioner såsom alarmer og nedtælling.
- Kalender med oversigt over opgaver/aftaler med deadlines og hvornår de forfalder.
- Projektplanlægning fx Kanban board eller lignende med forskellige typer af opgaver.

Kom i gang:

- Dan grupper
- Opret repo (angiv link under elevfeedback på timens skemabrik i Lectio)
- Opsæt kodeprojekt
- Find evt. inspiration/hjælp (men husk egne overvejelser er vigtigst)
- Lav første udgave af [brugerhistorie](https://jacs-pro.bitbucket.io/Teori/brugerhistorier.html) eller specifikation
- Design vha. diagrammer

---

## Rammer

- Gruppearbejde: 2-3 personer.
- Arbejde skal holdes opdateret i repo.
- Tænk både over kodens dybde og omfang. (Inddrag så vidt muligt også tidligere begreber.)
    - Se evt. [fagets faglige mål](https://jacs-pro.bitbucket.io)

## Logbog

I logbogen skal det primært dokumenteres hvilke programmeringsmæssige overvejelser fra udviklingen der kunne bruges i en [synopsis](https://jacs-pro.bitbucket.io/synopsis.html).

- Brugerhistorie (og hvor passer den i synopsis?)
- Design vha. diagram
- Synopsis: Metode
- Synopsis: Detaljeret kode
- Synopsis: Test af program
- Synopsis: Overordnet dokumentation
- Disposition til en præsentation (stikord til slides)
